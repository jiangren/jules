# 第6课：按钮开关 (Button Control)

这是我们第一次“告诉”Arduino要做什么，而不是让它自己在那儿跑。这是交互的开始！

## 🎯 目标
学习 `digitalRead` 读取外部信号，理解 `if/else` 判断逻辑。

## 🛠 硬件清单
*   Arduino Uno 主板
*   面包板
*   大轻触按键 x 1 (清单序号 33 或 32)
*   杜邦线

## 🔌 接线说明
我们使用 **INPUT_PULLUP (内部上拉)** 模式，这样接线超级简单，只需要两根线。

| 组件引脚 | 动作 | Arduino 接口 | 说明 |
| :--- | :---: | :--- | :--- |
| **按键脚 1** | -> | **Pin 2** | 信号线 |
| **按键脚 2** | -> | **GND** | 对角线那一端的引脚，接地 |

*(通常四脚按键是两两相通的，如果不确定，试着接对角线肯定没问题)*

## 🧩 玩耍指南 (给小朋友)
1.  **动作**：按下按钮。
2.  **现象**：灯亮了！松开手，灯灭了。
3.  **比喻**：“这就像门铃一样，按住才会响（亮）。”

## 💻 代码解析 (给大人)

### 核心语法
1.  **`pinMode(pin, INPUT_PULLUP)`**:
    *   开启 Arduino 内部的“上拉电阻”。
    *   **重要逻辑**：因为内部连到了5V，所以**不按的时候是 HIGH**，**按下的时候连通GND变成 LOW**。这叫“负逻辑”。
2.  **`digitalRead(pin)`**:
    *   Arduino 的“眼睛”，看一看这个引脚现在是有电(HIGH)还是没电(LOW)。
3.  **`if (条件) { ... } else { ... }`**:
    *   如果是真的，做这件事；否则，做那件事。

### 常见问题
*   **反了？** 如果你发现不按灯亮，按了灯灭。那就是逻辑反了。因为我们用了 Pullup，按下是 LOW。要在代码里写 `if (buttonState == LOW)` 才是“按下”。
