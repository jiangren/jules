## 2024-10-24 - Semantic Navigation and Focus States in H5 Games Menu
**Learning:** In the `games` project, game links were wrapped in generic `<div>` containers which impairs screen reader navigation, and CSS Module buttons lacked explicit keyboard focus states, making them inaccessible for keyboard users due to masked default browser outlines.
**Action:** When designing menus or collections of navigation links in web projects, always use semantic HTML tags like `<nav>` equipped with appropriate `aria-label`s (in Chinese to match the platform's primary language, e.g., `aria-label="游戏菜单"`). When styling custom buttons or links using CSS Modules (.less), always include explicit `&:focus-visible` styles (e.g., high-contrast outlines) to ensure keyboard navigation accessibility.
