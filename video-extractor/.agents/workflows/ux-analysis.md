# UX Animation Analysis (Slash Command)

此 workflow 会自动调用 `UX Animation Analysis` 技能，为你分析图片连帧并输出专业的动效分析结果。

## 执行步骤
1. 解析用户是否在命令后跟了需要分析的**目录路径** (例：`/ux-analysis ./my-frames`)。如果没有指定路径，请询问用户图片存放的位置。
2. 自动调用 `list_dir` 和 `view_file` 查阅这个目标目录下的相关图片，特别是那些刚才由 `extract` 命令生成的系列截图。
3. 扮演极具经验的高级交互设计师与转化率(CRO)专家角色，串联起刚刚看到的画面，在脑海中对这个界面的过场、留存、交互的重点进行细致推演。
4. 使用 `write_to_file` 创建一个名为 `ux-analysis.md` 的文件，将其放置在上述指定的图片同级目录下。内容需按照以下大纲输出给用户：
   - 📋 **页面全局扫描**：场景推演、当前交互流程优劣势分析。
   - 🔎 **现有动效/交互识别**：帧与帧之间现有动画效果的推导记录。
   - 🎨 **UI 视觉排版优化**：影响阅读和信任感的地方，如何整改布局。
   - 🪄 **微动效与 CRO 高级强化建议**：核心 CTA 按钮或特定场景如何加入极富巧思的互动（微交互动画、状态反馈）来提升点击欲和产品奢华感。
5. 告诉用户文件已生成，并且给出 1~2 句该界面最应该被重点关注的致命问题。
